# 会話の日記

LINE風のトーク履歴を貼り付けると、その中から「物語になりそうな小さな出来事」を見つけ、文章量と内容に合わせて見た目が変わる物語シールとして残す静的Webプロトタイプです。

完成版では本物のAIを使う想定ですが、今回は動く体験を優先し、AI APIは使っていません。ブラウザ上のJavaScriptだけで、キーワード判定とテンプレート生成によって候補を作ります。

## 体験すること

日記に書くほどではなかった会話を、小さな物語として切り出し、シールプリンターやシール用紙で印刷してノートに貼って集める体験を試します。

会話の内容と本文の長さに合わせて、シールプレビューの形、色、文字サイズが変わります。

1. LINEトークを貼る
2. 小説候補をつくる
3. 気に入った物語を選ぶ
4. 文章量に合う物語シールとして残す

## 使用例

以下は公開用に作った、完全に架空のトーク例です。実在の人物、学校、場所、過去の会話は使っていません。

```text
2026/06/18(木)
13:10 みお 今日のレポートどうする？
13:12 はる 参考文献探してるけど全然決まらない
13:14 みお AIに聞いたらそれっぽいの出たけど、リンク開けなかった
13:15 はる 便利なのか不便なのか分からんね
13:18 みお 最後は自分で確認するしかないやつ
13:20 はる それも含めて課題って感じする

2026/06/25(木)
09:02 そら 今日って泊まりの準備いる？
09:05 なな いると思う！タオルと歯ブラシは持った
09:09 そら 荷物もういっぱいなんだけど
09:12 なな 雨も降ってるし最悪だね
09:30 そら まだ家出てないのに帰りたい
13:40 なな 授業終わったらどこ集合する？
13:42 そら 出口の近くで待ってる
13:45 なな 了解、あとで笑い話にしよ

2026/06/27(土)
18:20 りく 今どこにいる？
18:21 まお 駅の近く！
18:22 りく 迎えに行こうか？
18:23 まお それはかなり助かる
18:25 りく じゃあ少し待ってて
18:26 まお ありがとう、今日いちばん安心した
```

この例からは「参考文献」「荷物」「迎え」の候補が出るようにしています。

## できること

- LINE風トーク履歴のテキスト貼り付け
- 架空サンプル会話の自動入力
- キーワード判定による候補生成
- 候補カードの選択
- 日付、題名、本文の編集
- 編集内容のシールプレビューへの即時反映
- 内容と文章量に合わせた shape / theme / 文字サイズのAI風おすすめ表示
- shape と theme の手動変更
- 「印刷する」「シールとして残す」ボタンからブラウザの印刷画面を開く
- 印刷時は入力欄や候補一覧を隠し、物語シールだけを出す
- Webリンクらしき文字列を「リンク」に置き換える簡易処理

## shape / theme

候補を選ぶと、まず本文の文字数を見て、無理なく読める形を優先します。そのうえでタイトルや本文のキーワードからシールの色をおすすめします。外部AI APIは使わず、JavaScriptのルールだけで判定しています。

- 281字以上: 長文用の縦長シールを優先
- 161〜280字: カード型または縦長を優先
- 71〜160字: 内容に合えば、カード型、丸型、しずく型、チケット型を使用
- 70字以下: 一言メッセージならラベル型も使用
- 雨、不安、夜、待つ: 短めならしずく型、長めなら縦長
- 迎え、安心、ありがとう: 短めなら丸型、長めならカード型または縦長
- 集合、イベント、予定: 短めならチケット型、長めならカード型

おすすめされた形、色、文字サイズは自動でシールに反映されます。あとからセレクトボックスで形と色を手動変更することもできます。

## 使い方

### PCで直接開く方法

1. このフォルダを開く
2. `index.html` をダブルクリックする
3. ブラウザでページが開く
4. 「サンプルを入れる」を押す
5. 「小説候補をつくる」を押す
6. 候補をクリックして、物語シールを確認する
7. 必要なら日付、題名、本文、かたち、色テーマを編集する
8. 「印刷する」または「シールとして残す」を押す

`file://` で直接開いても動くように作っています。

### GitHub Pagesで公開する手順

1. GitHubで新しいリポジトリを作る
2. `index.html` と `README.md` をアップロードする
3. リポジトリの `Settings` を開く
4. `Pages` を開く
5. `Build and deployment` の `Source` を `Deploy from a branch` にする
6. `Branch` を `main`、フォルダを `/root` にして保存する
7. 表示された公開ページのアドレスを開く

GitHub Pagesは、GitHub上のHTMLファイルをWebページとして公開できる機能です。

## 仕組み

このプロトタイプではAI APIを使わず、次のようなキーワードを見て候補を作っています。

- レポート、参考文献、リンク、課題
- 雨、荷物、泊まり、タオル、歯ブラシ
- 迎え、待ってて、安心
- 出席、遅刻、2限
- ゼミ、説明会、卒論
- 集合、駅、会場

さらに、できた本文の文字数を見て、shape / theme / 文字サイズを決めています。

- 長文: `long` を優先し、本文が見切れないようにする
- 中くらいの文章: `card` を中心にする
- 短文: 内容に応じて `label`、`round`、`drop`、`ticket` を使う
- しずく型や丸型は、文章が多い場合には使わない
- 文字数に応じて `text-large`、`text-medium`、`text-small` を切り替える

候補が見つからない場合でも、汎用候補として「小さな会話」を1件出します。

## プライバシーについて

このプロトタイプでは、会話をそのまま保存せず、名前や場所をぼかして物語化する想定です。

現在の実装では、入力内容をサーバーに送信しません。ブラウザの中だけで処理します。
